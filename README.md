## 有什么用

观察者和被观察者之间是抽象耦合的，不管是增加观察者还是被观察者都非常容易扩展。

根据单一职责原则，每个类的职责是单一的，那么怎么把各个单一的职责串联成真实的复杂的逻辑关系呢，观察者模式可以起到桥梁作用。 

观察者模式是松耦合的典型。 

在Android源码中，其中一个经典的使用到观察者模式的就是Android控件的事件监听模型。
